Findings to implement
- Home page is marked as a client component, causing mostly static landing sections to enter the client bundle.
- Root layout and home page both render AuroraBackground, duplicating a fixed decorative layer.
- Root canonical metadata currently points to "/" and child pages do not define their own canonical URLs.
- Public pages have useful descriptions but no shared route metadata helper or structured data.
- Sitemap uses the current timestamp for every request, which can make all pages look freshly modified.
- Robots configuration should explicitly disallow auth/account/login/private app surfaces while allowing public crawlable pages.
- SessionConsole has implemented state and API clients for replica/synthesis, but submit flow stops after initial agent responses.
- @supabase/ssr is unused.
- @supabase/supabase-js is only referenced by unused client factory helpers.
- components/agents/AgentAvatar.tsx and components/agents/AgentPortrait.tsx are unused.
- lib/db/repos/* and lib/db/types.ts are unused by the application; live persistence uses lib/chat/server.ts via Supabase REST.

Implemented changes
- Created SEO.txt continuity ledger.
- Converted the homepage from a full client component to server rendering with small client islands for auth redirect, CTA routing, section dots, reveal/typewriter effects.

- Added reusable SEO metadata helper and per-route canonical metadata for public and private pages.
- Added JSON-LD structured data for Organization, WebSite, and SoftwareApplication on the homepage.

- Removed unused @supabase/ssr and @supabase/supabase-js dependencies.
